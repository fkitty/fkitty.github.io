Class组件 VS 函数式组件

发表于 2020-05-17 | 更新于: 2020-05-17

基本写法

Class组件

使用箭头函数造成的缺陷
是一种语法糖，每次都会在构造函数中生成一个this.x = () => {}只要有一个实例，就有一个x,如果有很多个实例，就会有生成很多的x方法，这很多的x每个this是不同的实例，就会造成内存浪费

x(){}这种形式中的x是在原型上的，所有实例公用一个x方法

函数式组件

比较总结

  • 明显特征：
    • Class组件是有状态的(state),可以在一个对象上不停的对一个变量进行赋值
    • 函数式组件是无状态的(useState，16.8.0及其以上）看似有状态，实际上是没有的，因为DOM并没有改变，每次得到一个新的状态，之前的状态就不要了，就会有一个新的作用域生成，变量对应的内存发生了改变，类似迭代
  • 回归函数式和类本身
    • 函数组件可以方便的实现函数式（函数式本质：不能进行第二次赋值const）

数组方法总结（一）

发表于 2020-04-17 | 更新于: 2020-04-20

数组中是否存在某个元素

  • includes

  • indexOf

    indexOf：若元素存在，返回的是元素在数组中第一次出现时候的下标，元素不存在，则返回-1
    includes：若元素存在返回的是true,不存在是false
    参数说明：两个方法均可接受两个参数，第一个参数是查找的元素，第二个参数是从哪个位置开始查找（不传，默认的是从第一个元素开始，若有依次是（0，1，2，…），若小于0，-1表示最后一个元素，-2表示倒数第二个，以此类推）

根据检索条件查找元素

  • find

    根据检索条件，找到第一个符合条件的元素，若找不到返回undefined，不会改变数组

  • findIndex

    同find, 若找到返回的是该元素的下标，若找不到，返回的是-1

用Hexo+Github搭建博客

发表于 2020-04-07 | 更新于: 2020-04-07

环境

  • 注册github账号
  • 安装node
  • 安装git

步骤

  1. github新建仓库，仓库名必须是yourname.github.io的格式，选择使用”Initialize this repository with a README”, 点击seeting设置，在GitHub Pages下选择一个主题
  2. 本地新建一个文件夹，用来存放博客文件
  3. 打开git,目录切换到第二步新建的文件夹下，安装hexo,输入命令：npm i hexo-cli -g,mac下如果报错尝试用sudo npm i hexo-cli -g,安装完成之后使用hexo -v验证
  4. 初始化
    hexo init
    npm install
    hexo g
    hexo s
    打开localhost://4000本地查看
  5. 将hexo连接github
  • 打开存放博客的根目录下的 _config.yml
  • 修改配置
  1. 写文章、发布文章
  • 安装一个扩展npm i hexo-deployer-git, 第一次需要，后面就不需要了
  • 新建一个markdown文件 hexo new post "文章标题"
  • 打开博客目录下的source/_posts目录，这里列出的.md文件就是你的文章，找到新建的文章，编写.md文件，可以在编写的过程中本地运行，随时查看预览
  • 编写完之后，若编写过程中没有本地预览，可以使用：生成静态网页hexo g，hexo s，进行本地查看
  • 上传到github hexo d(在仓库下可以添加描述，注明预览链接)
