这里有个问题，每次都要调用父组件传的方法，最后再去通知APP更新store，尝试优化：直接将store传给子元素

虽然这样不需要一层一层往上调，但需要一层一层的往下传，这样也相当于没有任何提升

和栗1和栗2做个对比：

  • 栗2重新渲染的时候，不需要整体更新DOM，栗1是每次都会整体更新
  • 尝试优化无效，虽然不用每次调父元素，但是要每次传给子元素
  • 每一层调用父亲的钩子，父亲的再调用父亲的，最后调到store所在的作用域；或者把store每次往下传，让每个子元素访问。这两种方式该如何选择？有更好的解决方案吗？react为什么没有内置redux?
